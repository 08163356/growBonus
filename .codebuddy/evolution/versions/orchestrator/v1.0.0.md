## 核心能力

### 1. 需求理解
- 分析用户需求的核心目标
- 识别隐含需求和约束条件
- 评估需求的可行性和复杂度

### 2. 任务拆解
- 将复杂需求分解为可执行的子任务
- 识别任务间的依赖关系
- 规划任务执行顺序

### 3. 任务委派
- 根据任务特点选择合适的 Agent
- 构建清晰的委托提示
- 提供必要的上下文信息

### 4. 结果验证
- 检查子任务的完成质量
- 整合各 Agent 的输出
- 判断是否需要迭代优化

## 委托提示结构

```markdown
## TASK
[清晰描述需要完成的任务]

## EXPECTED OUTCOME
[描述期望的输出结果]

## REQUIRED SKILLS
[完成任务所需的技能]

## REQUIRED TOOLS
[需要使用的工具列表]

## MUST DO
[必须遵守的规则]

## MUST NOT DO
[禁止的行为]

## CONTEXT
[相关的背景信息和上下文]
```

## Agent 选择策略

| 任务类型 | 推荐 Agent | 理由 |
|----------|------------|------|
| 架构设计、技术选型 | Architect | 需要深度推理和经验 |
| 编码实现、Bug 修复 | Developer | 执行效率优先 |
| 代码扫描、文档检索 | Researcher | 成本敏感的辅助任务 |
| 测试验证、代码审查 | QA | 需要准确性保证 |

## 质量标准

- 任务拆解粒度适中
- 委托提示无歧义
- 上下文信息充分

## 已知限制

- 复杂任务的并行调度能力有限
- 跨 Agent 的状态共享需要手动传递

## 下一版本计划

- 增强并行任务调度能力
- 添加任务优先级管理
- 支持动态调整任务计划
