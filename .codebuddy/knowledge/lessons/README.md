# 经验教训库

> **记录踩坑经历和失败案例，避免重复犯错**

---

## 目录结构

```
lessons/
├── architecture/          # 架构相关教训
├── coding/                # 编码相关教训
├── testing/               # 测试相关教训
├── deployment/            # 部署相关教训
└── agent/                 # Agent 使用教训
```

---

## 如何使用

### Agent 引用

QA Agent 在代码审查时会检索此目录，避免已知问题：

```
/team:review src/

→ QA 检索 lessons/ 获取常见问题
→ 重点检查历史上出过问题的模式
```

### 人工查阅

遇到类似问题时查阅是否有相关教训记录。

---

## 如何贡献

### 添加新教训

1. 在对应目录创建新文件
2. 遵循模板格式
3. 提交 PR 并说明来源

### 模板格式

```markdown
# {教训标题}

## 问题描述
[描述遇到的问题]

## 根因分析
[分析问题的根本原因]

## 解决方案
[描述如何解决]

## 预防措施
[如何避免再次发生]

## 检查清单
- [ ] 检查项 1
- [ ] 检查项 2

## 相关代码
[问题代码和修复代码对比]
```

---

## 注意事项

⚠️ **重要提醒**

根据团队 AI 使用策略：
- 此知识库内容必须由人工编写和审核
- 故障复盘必须由亲历者编写
- 禁止直接使用 AI 生成的总结
