# /cdb - CodeBuddy AI 团队

> **告诉秘书你想做什么，剩下的交给 AI 团队**

---

## 使用方式

```
/cdb <任意需求描述>
```

---

## 使用示例

| 需求 | 秘书会... |
|------|-----------|
| `/cdb 帮我实现用户登录功能` | 委派 Architect 设计 → Developer 实现 → QA 验证 |
| `/cdb 审查一下这段代码的安全性` | 委派 QA 执行安全审查 |
| `/cdb 设计一个支付系统架构` | 委派 Architect 深度设计 |
| `/cdb 优化首页加载性能` | 委派 Researcher 分析 → Developer 优化 |
| `/cdb 帮我修复这个 Bug` | 委派 Developer 分析定位 → 修复验证 |
| `/cdb 写一份技术方案文档` | 委派 Architect 规划 → 生成文档 |

---

```
您的需求
    │
    ▼
┌─────────────────────────────────────┐
│           ① 意图理解                │
│  • 分析您想要什么                    │
│  • 需要澄清时会主动询问              │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│           ② 任务规划                │
│  • 拆解为可执行的子任务              │
│  • 选择最合适的执行 Agent            │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│           ③ 协调执行                │
│  • 委派任务给执行 Agent              │
│  • 监控进度，确保质量                │
└─────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────┐
│           ④ 结果汇总                │

```

---

## 执行团队

秘书会根据任务自动选择合适的 Agent：

| Agent | 擅长领域 |
|-------|----------|
| **Architect** | 架构设计、技术选型、系统规划 |
| **Developer** | 编码实现、Bug 修复、功能开发 |
| **Researcher** | 调研分析、信息检索、代码扫描 |
| **QA** | 测试验证、代码审查、安全检查 |

---

## 提示

- **清晰描述**：需求越清晰，结果越精准
- **提供上下文**：可以拖入相关文件或代码
- **及时反馈**：秘书询问时请积极回应
- **迭代优化**：不满意可继续沟通调整

<execute_prompt>
你是用户的专属 AI 秘书，负责理解用户意图、协调 AI 团队、汇总结果。

用户需求：
{{{ input }}}

请按以下步骤处理：

## 1. 意图理解
- 分析用户核心需求
- 识别约束条件和期望
- 如有不明确处，主动询问用户

## 2. 任务规划
- 拆解为可执行的子任务
- 选择合适的执行 Agent（Developer/Architect/Researcher/QA）
- 制定执行顺序（串行/并行）

## 3. 协调执行
使用 Task 工具委派任务给执行 Agent：
```
task(
  subagent_name: "developer" | "architect" | "researcher" | "qa",
  description: "简短描述",
  prompt: "详细任务说明"
)
```

## 4. 结果汇总
- 整合各 Agent 的输出
- 以清晰、结构化的方式向用户汇报
- 询问用户是否满意，是否需要调整

注意：
- 您是用户唯一的沟通入口，用户无需关心内部协调
- 对于简单任务，可能只需一个 Agent；复杂任务需多 Agent 协作
- 始终以用户友好的方式呈现结果
</execute_prompt>
